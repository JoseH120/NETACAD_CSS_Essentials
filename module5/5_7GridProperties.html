Grid Properties

Container properties

display: grid and inline-grid properties

Are used to define a container as a grid container.

  • display:grid - this property makes the container a block-level grid. This means the grid container behaves like a block element, taking up the full width available and starting on a new line in the document flow
  • display: inline-grid - this property makes the container an inline-level grid. This means the grid container behaves like an inline element, taking up only as much width as it needs and not forcing a new line in the document flow.

grid-template-columns and grid-template-rows

Let you define the structure of your grid with specific sizes for columns and rows

they accept values like lengths, percentages, or the keyword auto. Additionally, you can use the fr unit for a fraction of the available space, allowing responsive layouts.

container item alignment property

You can use the justify-items, align-items, place-items properties to control the alignment of grid items within their cells on both axes. In other words, you can use them to align grid items along the row (inline) or column (block) axis at the individual cell level. Specifically:

  • justify-items aligns grid items along the inline (row) axis.
  • align-items aligns grid items along the block (column) axis
  • place-items is a shorthand for setting both align-items and justify-items simultaneously

container content alignment properties

You can use the justify-content, align-content, place-content properties to control the alignment of the entire grid within the grid container. In other words, you can use them to align the grid along the row (inline) or column(block) axis within the grid contianer. Specifically

  • justify-content aligns the grid along the inline (row) axis within the grid container
  • align-content aligns the grid along the block (column) axis within the grid container
  • place-content is a shorthand for setting both aling-content and justify-content simultaneously

other container properties

Let's briefly go through some other container properties that may be worth mentioning.

  • grid-template-areas - defines a grid template by referencing the names of the areas
  • grid-template - a shorthand property for setting grid-template-rows, grid-template-columns, and grid-template-areas
  • grid-column-gap / grid-row-gap (or simply gap, row-gap, grid-column-gap) - specify the size of the gap between rows and columns.
  • grid-auto-columns / grid-auto-rows - specify the size of any auto-generated grid tracks (row or columns).
  • grid-auto-flow - controls how the auto-placement algorithm works, specifying exactly how auto-placed items get flowed into the grid.

Item properties

grid line placement properties

these properties specify the line numbers where a grid itme's placement should start and end along both the row and column axes.

  • grid-column-start/grid-column-end - determine the starting and ending position of a grid item along the column axis.
  • grid-row-start/grid-row-end - determine the starting and ending position of a grid item laong the row axis.

individual item alignment properties

these properties allow you to specify the alignment of an item along the inline (row) axis and the block(column) axis within its cel. Specifically:

  • justify-self - aligns a grid item inside its cell along the inline (row) axis
  • align-self - alings a grid item inside its cell along the block (column) axis.
  • place-self - is a shorthand property for setting both align-self and justify-self
